# Dashboard Wireframe Notes

## Purpose

This document describes low-fidelity page structure for a future internal review dashboard. It is not UI code.

中文描述：
本文档描述未来内部审核 dashboard 的低保真页面结构。它不是 UI 代码。

## Overview Dashboard

Suggested sections:

中文描述：
建议区域：

- Brand card
- Current phase card
- Validation status card
- FAQ pipeline card
- Shopify package card
- Risk summary card
- Next human actions card

The overview should be dense, scannable and operational rather than marketing-style.

中文描述：
Overview 应该信息密集、易扫描、偏运营工具风格，而不是营销页面风格。

Suggested sections:

中文描述：
建议区域：

中文描述：

## Safety Boundaries Page

Suggested sections:

中文描述：
建议区域：

- No Shopify API
- No auto-publish
- No live theme modification
- No final page creation
- No FAQPage Schema
- No Product Schema
- No claim validator bypass
- No source status bypass

## Visual Tone

The dashboard should feel like a calm internal operations tool.

中文描述：
Dashboard 应该像一个安静、清晰的内部运营审核工具。

Recommended UI direction for a future phase:

中文描述：
未来阶段建议 UI 方向：

- compact cards
- restrained colours
- status badges
- file preview panels
- risk tables
- checklist rows
- no decorative landing-page hero
- no public SaaS marketing layout
