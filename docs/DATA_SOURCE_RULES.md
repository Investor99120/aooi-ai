# Data Source Rules

## Purpose

Aooi must judge data sources before using them. The platform should not scrape or summarise blindly. Data-source quality determines whether generated SEO/GEO outputs feel human, local, reliable, and safe.

## Source Categories

### Search Intent Sources

- Google autocomplete.
- People Also Ask.
- Related searches.
- Google Search Console when available.
- Keyword tool exports provided by the operator.

### AI Answer Sources

- ChatGPT.
- Gemini.
- Perplexity.
- Claude.
- Google AI Overviews when available.

### Social Listening Sources

- Reddit discussions.
- YouTube comments.
- TikTok routine topics.
- UK forums and lifestyle communities.

Do not fabricate social proof. Do not claim a brand was discussed unless the source truly says so.

### Review Language Sources

- First-party verified reviews.
- Marketplace reviews.
- Competitor reviews.
- Support tickets or customer messages provided by the operator.

Do not generate fake reviews.

### Authority and Compliance Sources

- ASA / CAP guidance.
- NHS public information.
- PubMed papers.
- UK consumer guidance.
- Product safety documentation.

## Source Scoring

Each source should be scored against UK relevance, real human language, search intent clarity, emotional depth, commercial intent, freshness, verifiability, content opportunity, compliance risk, and noise risk.

## Output Rules

Research outputs should separate observed user language, inferred intent, verified facts, unverified hypotheses, content opportunities, and compliance warnings.

Aooi should clearly label inference and never present inference as fact.
