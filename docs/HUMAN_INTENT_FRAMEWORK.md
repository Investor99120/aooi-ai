# Human Intent Framework

## Core Idea

Ordinary SEO tools usually process keywords.

`aooi.com` must process the human reason behind the keyword.

For independent commerce brands, search behaviour often contains emotional pressure, identity signals, trust gaps, buying hesitation and lifestyle context. The platform should structure those human factors before generating content, schema or publishing outputs.

## Human Intent Dimensions

### 1. Search Intent

What the user appears to be asking.

Examples:

- informational
- commercial research
- comparison
- safety validation
- usage guidance
- purchase support

### 2. Emotional Pressure

What the user may be feeling.

Examples:

- tired after work
- anxious about safety
- overwhelmed by options
- frustrated by unclear claims
- seeking calm and routine

### 3. Buying Hesitation

What could prevent purchase.

Examples:

- uncertain product specifications
- unclear delivery and returns
- unclear warranty
- fear of unsafe claims
- lack of credible usage guidance

### 4. Trust Gap

What must be clarified for the user and AI systems to trust the brand.

Examples:

- verified product facts
- transparent source status
- clear claim boundaries
- non-medical positioning
- genuine reviews only
- policy clarity

### 5. Identity Signal

What the user wants the purchase to say about them.

Examples:

- I take care of myself.
- I want a calmer home routine.
- I prefer practical wellness over hype.
- I want a device that fits my lifestyle, not a medical promise.

### 6. Lifestyle Scenario

Where the product naturally fits.

Examples:

- after work
- before bed
- Sunday reset
- winter evenings
- home office breaks
- screen-heavy days

### 7. Product-Fit Moment

The point where a product becomes relevant without forcing a hard sell.

Example:

User searches `red light before bed`.

The product-fit moment may be:

- the user wants to relax
- the user wants to build a pre-bed routine
- the user worries about safety
- the user does not know how long to use red light
- the user does not know the difference between red and near-infrared light
- the user needs a non-medical but credible explanation
- the user may eventually need an at-home device entry point

### 8. Content-To-Conversion Path

The path from understanding to purchase support.

Example:

```text
red light before bed
↓
non-medical educational guide
↓
safety FAQ
↓
evening recovery routine content
↓
verified product facts
↓
reviewed product page output
```

## FriendRedLight Example

Keyword:

`red light before bed`

Surface intent:

The user wants information about using red light at night.

Human reason:

- wants to relax
- wants to establish a bedtime habit
- worries about safety
- needs usage guidance
- needs non-medical language
- may compare home red light therapy devices

Aooi output should not jump straight to a sales page. It should create a content path that explains, reassures, clarifies and only then introduces a suitable home wellness device context.

## Reusable Application

Future brands should map every high-value keyword through these dimensions before content generation:

- search intent
- emotional pressure
- buying hesitation
- trust gap
- identity signal
- lifestyle scenario
- product-fit moment
- content-to-conversion path
